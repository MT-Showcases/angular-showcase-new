🎓 Perché il Service Facade Pattern?

In Angular, la regola d'oro è: i componenti gestiscono la UI, i services gestiscono i dati. Il pattern Service Facade va un passo oltre: il service nasconde completamente i dettagli dell'API ai componenti. Se l'API cambia, modifichi solo il service.

✅ Con Service Facade

// Componente — semplice e testabile
private posts = inject(PostsService);
loadPosts() {
  this.posts.getPosts().subscribe(...);
}

❌ Anti-Pattern: HttpClient diretto

// Componente — accoppiato all'API, non testabile!
private http = inject(HttpClient);
loadPosts() {
  this.http.get<Post[]>('https://api/posts').subscribe(...);
}

⚠️ Se l'URL cambia, devi modificare ogni componente. Se aggiungi caching, lo devi aggiungere ovunque. Non testabile senza mock di HttpClient.

🚫 Anti-Pattern: Subscribe senza cleanup

Gli Observable HTTP sono one-shot (si completano dopo la risposta), quindi non richiedono unsubscribe. Ma attenzione agli Observable infiniti (es. interval, WebSocket):

// ❌ ANTI-PATTERN — memory leak con Observable infiniti
ngOnInit() {
  interval(1000).subscribe(v => this.counter = v); // ← mai completato!
}

// ✅ CORRETTO — usa takeUntilDestroyed() o async pipe
counter$ = interval(1000).pipe(takeUntilDestroyed());
// In template: usa async pipe con counter$

→ Approfondisci nel Pattern Explorer il pattern Observer e come gestire correttamente i lifecycle degli Observable in Angular.

🏛️ Cos'è un Service Facade?

📦 Separazione delle Responsabilità

Il componente si occupa solo della UI. Il service gestisce tutta la logica HTTP e i dati.

♻️ Riusabilità

Il service può essere usato da più componenti. Scrivi la logica HTTP una volta sola.

🧪 Testabilità

Facile creare mock del service. Testa UI e logica separatamente.

🔧 Manutenibilità

Se l'API cambia, modifichi solo il service. Tutti i componenti continuano a funzionare.

@if (error()) {
⚠️ {{ error() }}
} @if (loading()) {

}

✨ CREATE - POST Request

Crea un nuovo post inviando dati al server. Il service usa http.post()

📚 READ - GET Request

Recupera dati dal server. Il service usa http.get()

✅ Vantaggi del Service Facade Pattern

🎯

Single Responsibility

Ogni classe ha un solo scopo: il componente gestisce la UI, il service gestisce i dati.

🔒

Encapsulation

I dettagli dell'API (URL, headers, parsing) sono nascosti nel service.

⚡

Performance

Puoi aggiungere caching, debouncing, retry logic nel service senza toccare i componenti.

🔄

Consistency

Tutti i componenti usano la stessa logica HTTP. Nessuna duplicazione o inconsistenza.
