📚 Perché il Data Binding?

Senza data binding dovresti scrivere: document.getElementById('myEl').textContent = value ad ogni aggiornamento — codice fragile e difficile da testare.

Angular gestisce questo automaticamente tramite il suo Change Detection: quando una proprietà del componente cambia, il DOM viene aggiornato in automatico. ✨

→ Approfondisci i pattern di stato nel Pattern Explorer

1. Interpolazione {{ "{{ }}" }}

Mostra valori e risultati di espressioni nel DOM. Flusso dati: Componente TypeScript → DOM (sola lettura)

Variabile:
{{ text }}
{{ text }}
Metodo:
{{ getText() }}
{{ getText() }}
Con parametri:
{{ getDynamicText('esempio') }}
{{ getDynamicText('esempio') }}
Calcolo:
{{ sum(10, 25) }}
{{ sum(10, 25) }}

⚠️ Anti-pattern: metodi con side-effects nell'interpolazione

{{ '{{' }} saveUserToDatabase() {{ '}}' }}

Problema: i metodi nel template vengono chiamati ad ogni ciclo di Change Detection (potenzialmente centinaia di volte al secondo). Un metodo con side-effects (chiamate HTTP, log, mutations) causerebbe caos.

Soluzione: usa un getter per valori derivati, o un pipe per trasformazioni riutilizzabili.

2. Property Binding [ ]

Lega proprietà HTML/DOM a valori del componente. Usa sempre [attr] per attributi DOM, non l'interpolazione!

Path: {{ imgLogoSrc }}
[src]="imgLogoSrc"

⚠️ Anti-pattern: usare l'interpolazione per gli attributi

<img src="{{ '{{' }} imgLogoSrc {{ '}}' }}">

Problema: il browser fa una richiesta HTTP con il literal {{ '{{' }} imgLogoSrc {{ '}}' }} prima che Angular lo sostituisca, causando un errore 404 temporaneo e un potenziale flash di contenuto rotto.

Soluzione: usa sempre [src]="imgLogoSrc" per attributi che contengono URL, percorsi, o valori dinamici.

3. Event Binding ( )

Ascolta eventi DOM e chiama metodi del componente. Flusso dati: DOM Event → Componente TypeScript (azione)

(click)="onClick()"

⚠️ Anti-pattern: logica inline nell'event binding

(click)="imgLogo = '/path.png'; count++; saveToStorage()"

Problema: logica complessa nel template è difficile da testare, leggere, e riutilizzare. Il template dovrebbe descrivere la struttura, non la logica.

Soluzione: sposta tutta la logica nel componente TypeScript ((click)="onClick()") e mantieni i metodi piccoli e testabili.
